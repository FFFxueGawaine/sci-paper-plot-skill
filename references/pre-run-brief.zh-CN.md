# 运行前提示单

这个文件用于约束 Codex 在运行绘图程序前的行为。核心原则是：**只读命令可以直接运行；会写文件、复制 demo、生成图片或替换真实数据的命令，先向用户获取提示**。

## 1. 哪些命令可以直接运行

这些命令只读或只打印信息，一般不需要额外确认：

```bash
python scripts/scimplstyle_mssp_cli.py beginner-guide --lang zh
python scripts/scimplstyle_mssp_cli.py run-brief --lang zh
python scripts/scimplstyle_mssp_cli.py recommend "误差分布" --lang zh
python scripts/scimplstyle_mssp_cli.py list-demos
python scripts/scimplstyle_mssp_cli.py style-guide
```

## 2. 哪些操作运行前要先问用户

下面这些操作会写文件、复制文件或生成图片，运行前要先确认提示单：

- `copy-demos`
- `preview-gallery`
- `check-demos`
- 任何真实绘图脚本
- 任何把占位数据替换成真实数据的脚本
- 任何批量生成论文图的任务

## 3. 最小提示单

如果用户没有给完整信息，先问下面 6 项：

| 信息 | 为什么需要 |
|---|---|
| 绘图目标 | 决定用折线、分布、热图、FRF、1x3 对比等哪类模板。 |
| 数据来源 | 判断先用占位数据，还是直接读取 CSV/TXT/NPY/Excel。 |
| 输出位置 | 避免把生成图写进 skill 安装目录。 |
| 语言和字体 | 决定英文 SCI、中文报告、宋体标题或中英混排。 |
| 子图标号位置 | 决定 `(a)(b)(c)` 放上方、左上角、下方或不放。 |
| 版面约束 | 决定单栏/双栏、几行几列、图例、色条和局部放大框。 |

## 4. 可以直接问用户的话

```text
在我运行绘图程序前，请先告诉我：
1. 你要画的图是什么目标？例如误差分布、时域响应、FRF、热图。
2. 你现在有真实数据吗？如果有，请给数据文件绝对路径；如果没有，我先用占位数据。
3. 输出脚本和图片放到哪个工作文件夹？我不会写进 skill 安装目录。
4. 图中文字用英文 SCI、中文报告，还是中英混排？中文标题是否用宋体？
5. `(a)(b)(c)` 标号放在哪里：上方居中、左上角、下方居中、左下方，还是不放？
6. 版面是单图、1x3、2x2、2x3，还是按参考图？
```

## 5. Codex 判断规则

- 如果用户已经明确给出目标、数据、输出位置和版面要求，可以先复述确认，再运行。
- 如果只缺一两项，只问缺失项，不要让用户重新填完整表。
- 如果用户说“先跑个 demo 看看”，可以默认使用占位数据，但仍要确认输出文件夹。
- 如果用户给的是安装目录，提醒他改到项目工作区。
- 如果用户说“我是新手”，先运行或总结 `run-brief --lang zh` 和 `beginner-guide --lang zh`，再进入绘图。
